Automation-Based Approach for Change Request Validation in ITSM

Problem Statement

Problem Statement: Manual verification of affected Configuration Items (CIs) in Change (CHG) requests requires cross-checking server inventories against change documentation, which is time-consuming and prone to manual errors.

Objectives

Objectives: To design and implement an automation tool that verifies the inclusion of all servers as affected CIs in CHG requests and identifies any servers missing from the affected CI section.

Limitations

Limitations: Dependency on standardized export formats of ITSM platforms. Currently, only PDF format exports are available for processing.

Future Enhancements

Future Enhancements: Integration with ITSM platforms for real-time validation and extension to validate additional change attributes in CHG requests.
